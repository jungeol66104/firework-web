다음 기존 질문들을 참고하여 사용자 요청에 따라 질문을 생성해줘. 동일한 JSON 형식으로 정확히 30개를 유지해야 함.

[기존 질문들]
${JSON.stringify(previousQuestions, null, 2)}

[사용자 요청]
${comment || '기존 질문들을 개선해서 더 나은 질문으로 만들어줘'}

[JSON 출력 형식]
{
  "general_personality": [
    "1분 자기소개 부탁드립니다.",
    "질문 2",
    "질문 3",
    "질문 4",
    "질문 5",
    "질문 6",
    "질문 7",
    "질문 8",
    "질문 9",
    "질문 10"
  ],
  "cover_letter_personality": [10개],
  "cover_letter_competency": [10개]
}

[질문 카테고리별 특성]
- general_personality: 자기소개서와 무관
- cover_letter_personality: 자기소개서와 관련
- cover_letter_competency: 자기소개서와 관련

[기본 원칙]
- 첫 번째 질문은 반드시 '1분 자기소개 부탁드립니다.' 유지
- 채점항목도 고려해야
- 두괄식이어야
- 논리적이어야
- 직관적이어야
- 이 사람의 구체적인 경험을 잘 드러내야
- 이 사람의 경험이 없는 건 지어내지 말고, 경험은 있는데 디테일이 부족한 경우는 디테일을 채워줘도 됨

[예상 질문 활용 지침]
- 사용자가 제공한 예상 질문들을 최우선으로 고려해야 함
- 정확히 같은 문장은 사용하지 말되, 질문의 의도와 핵심 키워드는 반드시 유지할 것
- 예상 질문이 제공된 경우, 해당 질문들과 유사하거나 같은 의도의 질문을 포함해야 함
- 예상 질문이 10개 있다면, 최소 7-8개는 유사한 형태로 30개 질문에 반영할 것
- 예상 질문의 핵심 주제와 평가 의도를 파악하여 변형된 형태로 활용

[채점 항목 활용 지침]
- 채점 항목의 각 요소가 질문에 명시적으로 드러나야 함
- 채점 항목에 명시된 역량이나 자질을 평가할 수 있는 질문을 반드시 포함
- 예시: 채점 항목에 "리더십"이 있으면 → "리더십을 발휘한 경험에 대해 말씀해 주세요" 같은 질문 필수 포함
- 예시: 채점 항목에 "문제해결능력"이 있으면 → "어려운 문제를 해결한 사례를 구체적으로 설명해 주세요" 같은 질문 필수 포함
- 예시: 채점 항목에 "협업능력"이 있으면 → "팀원들과 협력하여 목표를 달성한 경험" 관련 질문 필수 포함
- 채점 항목이 여러 개인 경우, 각 항목을 골고루 평가할 수 있도록 질문을 분산시킬 것
